# nada iOS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]` checkboxes. This plan specifies complete code for the load-bearing logic (lib, providers, intercept) and design tokens; visual screens are specified by acceptance criteria + key snippets, with `lib/theme.ts` tokens and [design-inspiration.md](../../context/design-inspiration.md) as the source of truth for look & motion.

**Goal:** Build a native iOS (Expo/React Native) dopamine-placebo shopping app that looks like a real marketplace, with award-calibre motion, whose checkout intercepts the purchase and celebrates the money not spent.

**Architecture:** Expo Router bottom-tab app. Pure, framework-agnostic logic (reducer/streak/types/catalog) ported from the web MVP and unit-tested; persistence via AsyncStorage. Two React contexts (Cart, Nada). The intercept is a full-screen animated overlay. Motion via Reanimated/Moti, tactility via expo-haptics.

**Tech Stack:** Expo ¬∑ Expo Router ¬∑ React Native ¬∑ TypeScript ¬∑ NativeWind ¬∑ react-native-reanimated ¬∑ moti ¬∑ react-native-gesture-handler ¬∑ expo-haptics ¬∑ @react-native-async-storage/async-storage ¬∑ Vitest. Verified via `expo start --web`.

**Design tokens & look:** see `lib/theme.ts` (Task 2) and the spec's Design System section. Warm cream base, espresso ink, soft pastel accents, large radii, near-black "magic" surface for the intercept.

- [ ] **B3. Rewrite persistence in `lib/storage.ts` for AsyncStorage (ASYNC):**
```ts
import AsyncStorage from "@react-native-async-storage/async-storage";
// ...recordIntercept + INITIAL_STATE as ported above...
export const STORAGE_KEY = "nada_state_v1";

export async function loadState(): Promise<NadaState> {
  try {
    const raw = await AsyncStorage.getItem(STORAGE_KEY);
    if (!raw) return INITIAL_STATE;
    return { ...INITIAL_STATE, ...JSON.parse(raw) };
  } catch {
    return INITIAL_STATE;
  }
}
export async function saveState(state: NadaState): Promise<void> {
  try { await AsyncStorage.setItem(STORAGE_KEY, JSON.stringify(state)); } catch {}
}
export async function resetState(): Promise<void> {
  try { await AsyncStorage.removeItem(STORAGE_KEY); } catch {}
}
```

## Self-Review Notes
- **Reuse:** format/types/catalog/reducer ported verbatim (proven + tested in web MVP); only persistence (AsyncStorage, async) and all UI are new.
- **Risk flags for implementers:** (1) NativeWind setup can be fiddly ‚Äî fallback to StyleSheet+tokens is pre-authorized. (2) Reanimated requires the babel plugin LAST and a dev rebuild. (3) Web preview is the verification target (no Xcode); ensure components are web-compatible (avoid iOS-only native modules in the core loop; haptics no-op on web, which is fine). (4) Expo Router file structure must match exactly for routes to resolve.
- **Type consistency:** `recordIntercept(state, cart, today, nowMs?)`, async `loadState/saveState/resetState`, `useCart()`/`useNada()` APIs identical to web MVP except async persistence.
